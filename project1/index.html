1. Pathname

A pathname is a string that represents the location of a file or directory in the file system. It can be either an absolute or a relative pathname.

An absolute pathname starts from the root directory / and specifies the complete path to a file or directory, while a relative pathname is specified in relation to the current working directory.

Usage (absolute path):
$ /home/user/Documents/file.txt

Example of relative path:

Usage:
$ Documents/file.txt

2. The . (Current Directory)

The . (dot) represents the current directory. It is often used in commands to refer to the directory you are currently in, or to execute programs in the current directory.

Usage:
$ ./script.sh

In the example above, the ./ tells the system to execute the script.sh file from the current directory.

3. The .. (Parent Directory)

The .. (double dot) represents the parent directory, or the directory one level above the current directory.

It is commonly used to navigate up the directory structure.

Usage:
$ cd ..
